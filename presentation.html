04 · התאמה למסלול
Track Alignment · תקשורת וסייבר

למה הפרויקט תואם למסלול תקשורת וסייבר

מערכת מבוזרת, מבוססת קונטיינרים, עם תקשורת בין-שירותית מאובטחת

🌐

תקשורת · Communications

  • פריסה מבוססת קונטיינרים : Docker multi-stage + docker-compose
  • תקשורת מוצפנת בלבד : פרוטוקול HTTPS/TLS בכל הערוצים, ללא HTTP
  • תקשורת בין-שירותית : REST מול 6+ שירותים חיצוניים
  • ארכיטקטורה מונחית-אירועים + Webhooks דו-כיווניים
  • Cloudflare Tunnel : חשיפה מאובטחת ללא פתיחת פורטים נכנסים
  • בדיקות עומסים : מדידת throughput וזמני תגובה תחת בקשות מקביליות
  • נקודת קצה /health לניטור liveness של הקונטיינר
🛡️

סייבר · Cyber Security

  • אימות והרשאות : Clerk עם בקרת גישה מבוססת תפקיד (RBAC)
  • אימות חתימת Webhooks (Svix) : מניעת זיוף בקשות נכנסות
  • בידוד דיירים : הפרדת נתונים מלאה בין ארגונים (tenant isolation)
  • הקשחת קונטיינר : משתמש non-root, alpine מינימלי, .dockerignore
  • ניהול סודות : מפתחות API ב-env_file, placeholders בזמן build
  • ולידציית קלט עם Zod בכל הגבולות : מניעת injection
  • ולידציית env בעליית המערכת (t3-oss/env) : מניעת תצורה שגויה
